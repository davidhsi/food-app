## Gotchas & working norms

- **Read `docs/` before probing tooling.** Facts are written down (`deployment.md`,
  `feature-timeline.md`, `decisions/`, `CLAUDE.md`).
- **`main` is production.** Any push deploys. Don't push half-finished work to `main`;
  use a branch/PR (gets a Preview URL) when unsure. For a polish pass, a branch + PR
  preview is the safer pattern than committing edits straight to `main`.
- **Sync before committing.** `git fetch` (+ merge/rebase) at the start and before
  pushing â€” `main` has diverged before when parallel sessions landed concurrently.
- **`NEXT_PUBLIC_SITE_URL` is build-time inlined** â€” changing it requires a *redeploy*.
- **Verify gate is `typecheck` + `build`** (no test runner by design) + a 375px visual
  pass. Lint isn't configured (it prompts interactive setup). Data ingest runs locally,
  never at build.
