## Gotchas & working norms for next session

- **Read `docs/` before probing tooling.** Ops/deploy/feature facts are written
  down (`deployment.md`, `feature-timeline.md`, `decisions/`, `CLAUDE.md`). Check
  there first.
- **`main` is production.** Any push deploys. Don't push half-finished work to
  `main`; use a branch/PR (gets a Preview URL) when unsure.
- **Sync before committing.** `main` diverged twice this session because parallel
  sessions/PRs were landing concurrently. `git fetch` (and merge/rebase) at the
  start and before pushing to avoid non-fast-forward surprises.
- **`NEXT_PUBLIC_SITE_URL` is build-time inlined** — changing it requires a
  *redeploy*, not just saving the env var.
- **Verify gate is `typecheck` + `build`** (no test runner by design), plus a
  375px visual pass. Data ingest runs locally, never at build.
